# JobCopilot - AI 面试训练工具 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品愿景
JobCopilot 是一个面向个人求职者的响应式 AI 面试训练工具，帮助用户围绕目标岗位方向系统提升面试表现。产品支持用户在电脑端和手机端完成从资料上传、岗位分析、模拟面试到深度复盘的完整闭环。

JobCopilot 以“岗位方向”为主要组织单位，例如技术、产品、运营、销售、管理、转岗/晋升等。用户可以围绕一个岗位方向长期训练，沉淀自己的高分回答、项目故事和岗位能力模型。

### 1.2 产品定位
**JobCopilot = 面向 C 端个人求职者的岗位方向面试训练工具。**

产品核心动作是：

1. 选择岗位方向。
2. 上传简历/JD 或补充目标岗位信息。
3. 获取岗位匹配分析与预测考点。
4. 开始 AI 模拟面试。
5. 查看复盘并沉淀高分表达。

### 1.3 目标用户
*   **正在准备面试的个人求职者：** 需要针对技术、产品、运营等岗位方向进行系统练习。
*   **转岗或晋升候选人：** 需要把过往经历转译成目标岗位认可的表达方式。
*   **需要高频脱敏训练的求职者：** 希望通过 AI 练习降低面试紧张感，提升结构化表达能力。
*   **移动端高频使用用户：** 希望在碎片时间通过手机上传资料、看分析、语音练习和复盘。

## 2. 产品发展方向

### 2.1 从“公司备战”调整为“岗位方向训练”
产品以岗位方向建立训练体系。因为这样更适合 C 端个人用户，也能降低使用门槛。

示例岗位方向：

*   技术岗
*   产品岗
*   运营岗
*   销售岗
*   管理岗
*   转岗/晋升
*   自定义方向

每个岗位方向下沉淀一套训练资产：

*   岗位能力模型
*   高频题库
*   面试追问逻辑
*   用户个人故事库
*   历史模拟面试记录
*   高分回答模板
*   能力趋势与训练进度

### 2.2 C 端个人工具优先
当前版本优先服务个人求职者，暂不建设团队协作、企业权限、教练管理后台等 To-B 功能。

### 2.3 桌面端与手机端都支持完整闭环
产品采用响应式 Web App 形态，电脑端和手机端都应支持完整核心流程：

*   上传简历/JD
*   查看分析报告
*   开始模拟面试
*   查看深度复盘
*   收藏高分回答
*   管理个人故事库和题库

两端区别主要在体验侧重点，而不是功能缺失。

**桌面端侧重点：**

*   深度准备
*   大屏分析报告
*   多栏沉浸式模拟面试
*   故事库、题库、资料管理
*   多次训练结果对比

**手机端侧重点：**

*   随时开始训练
*   文件选择、拍照或从手机文件系统导入简历/JD
*   卡片式分析报告
*   语音优先的单列模拟面试
*   快速查看复盘与收藏回答
*   5 分钟快速练习

## 3. 系统核心功能架构

系统整体采用响应式布局。桌面端使用左侧导航 + 主内容区的 SaaS 式布局；手机端使用底部导航 + 单列内容流，保证核心流程在小屏上也可以完整完成。

### 3.1 当前 v2 前端导航
当前 `jobcopilot_v2.zip` 中的 React 项目与 `JobCopilot_Figma_Export.html` 已提供以下 UI 原型导航：

1.  **首页 (Dashboard)：** 最近训练、方向入口、开始新练习。
2.  **模拟面试 (Interview)：** 准备、分析、面试、复盘四阶段工作流。
3.  **职位库 (Job Library)：** 管理目标 JD 与面试状态。后续产品上建议逐步收敛为“岗位方向/目标岗位资料”，避免变成重度公司 CRM。
4.  **简历库 (Resume Library)：** 管理多版本简历。
5.  **故事库 (Story Library)：** 沉淀个人经历和高分回答。
6.  **设置 (Settings)：** 管理 AI 面试偏好和个人资料。

### 3.2 目标版导航建议
后续 UI 设计与产品迭代建议逐步调整为：

1.  **首页：** 展示最近训练、岗位方向进度、快速开始入口。
2.  **岗位方向：** 管理技术、产品、运营、销售、管理等训练方向。
3.  **模拟面试：** 基于当前岗位方向开始完整训练流程。
4.  **故事库：** 沉淀个人经历、项目故事和高分回答。
5.  **题库：** 按岗位方向管理高频题、追问题和专项训练题。
6.  **复盘：** 查看历史训练报告、能力趋势和回答重塑结果。
7.  **资料：** 管理简历、JD、作品集等基础资料。
8.  **设置：** 配置个人信息、AI 面试官风格、语言和严格程度。

## 4. 岗位方向模块 (Role Tracks)

### 4.1 目标
帮助用户按目标岗位方向建立长期训练体系，而不是围绕单家公司临时准备。

### 4.2 功能描述
每个岗位方向包含：

*   **岗位能力模型：** 拆解该方向常见考察能力。例如技术岗关注项目深度、系统设计、代码能力、协作能力；运营岗关注数据分析、活动策划、用户增长、资源协调。
*   **训练进度：** 展示该方向下的练习次数、平均分、薄弱能力和近期提升趋势。
*   **高频题库：** 根据岗位方向生成常见问题、压力问题和追问问题。
*   **故事映射：** 将用户个人经历映射到该岗位方向下的能力要求。
*   **快速开始：** 一键进入该方向的模拟面试或专项训练。

### 4.3 交互逻辑
用户首次进入产品时，可以选择一个默认岗位方向。后续也可以新增或切换方向。所有面试记录、题库收藏、故事库内容都可以与一个或多个岗位方向关联。

## 5. 面试核心链路详述 (Interview Module)

面试模块是系统的核心，设计为“准备 -> 分析 -> 面试 -> 复盘”的四阶段工作流。电脑端和手机端都必须支持完整流程。

### 5.1 阶段一：准备阶段 (Upload Stage)
**目标：** 获取构建场景化、针对性面试所需的上下文。

**功能描述：**

*   选择当前训练的岗位方向。
*   上传简历，支持 PDF/DOCX/图片等格式。
*   输入或上传目标 JD，支持文本、文件、截图。
*   可选填写目标级别、工作年限、面试语言、训练强度等信息。

**桌面端体验：**

*   使用左右分栏或表单 + 上传区布局。
*   适合同时查看简历、JD 和岗位方向信息。

**手机端体验：**

*   使用单列步骤式流程。
*   支持从手机文件系统选择文件。
*   支持拍照上传纸质资料或截图。
*   支持粘贴 JD 文本。

**交互逻辑：**

简历和 JD 至少提供其一即可开始基础分析；两者都提供时，系统给出更完整的人岗匹配分析。按钮文案根据资料完整度区分为“开始基础分析”或“开始完整分析”。

### 5.2 阶段二：智能分析结果 (Analysis Stage)
**目标：** AI 根据简历、JD 与岗位方向，帮助用户建立面试准备认知。

**功能描述：**

*   **岗位匹配度评分：** 给出整体匹配度和维度拆分。
*   **核心优势：** 列出能支撑目标岗位的已有经历和能力证据。
*   **关键短板：** 指出简历或表达中缺失但岗位常考的能力点。
*   **预测面试焦点：** 生成 3-5 个高概率被追问的问题。
*   **训练建议：** 推荐进入完整模拟、专项训练或故事库补强。

**桌面端体验：**

*   展示完整报告，包括能力雷达、优劣势列表、预测问题和建议训练路径。

**手机端体验：**

*   使用卡片式分段展示：
    *   匹配分
    *   3 个优势
    *   3 个短板
    *   5 个预测追问
    *   开始模拟面试按钮

### 5.3 阶段三：模拟面试实战 (Interview Stage)
**目标：** 提供沉浸式、有压力、可反复练习的真实面试体感。

**功能描述：**

*   AI 根据岗位方向、简历和 JD 提问。
*   支持语音回答和文字回答。
*   支持严格模式，触发追问、压力测试和反事实质疑。
*   实时记录 AI 问题和用户回答。
*   标注当前问题正在考察的能力点。

**桌面端布局：**

*   **考点面板：** 显示当前考察能力、岗位要求和追问线索。
*   **AI 互动区：** 展示 AI 面试官状态、语音波形、录音控制和结束按钮。
*   **实时文字记录：** 展示多轮问答记录，方便回看。

**手机端布局：**

*   单列沉浸布局。
*   顶部显示当前考点和面试进度。
*   中间展示 AI 问题、语音波形和回答状态。
*   底部固定录音、暂停、结束按钮。
*   文字记录默认折叠，可展开查看。

### 5.4 阶段四：深度复盘与评估 (Review Stage)
**目标：** 不只是指出问题，而是提供可执行的改进框架。

**功能描述：**

*   **综合评分：** 展示整体表现、表达结构、岗位匹配度、业务深度、抗压能力等维度。
*   **关键问题回放：** 标记本次面试中最值得改进的回答。
*   **回答重塑：** 对用户原话进行案例级拆解。
*   **训练建议：** 推荐下一次练习方向或需要补强的故事素材。
*   **收藏与沉淀：** 用户可以把重塑后的高分回答保存到故事库或题库。

**回答重塑结构：**

1. 面试官的提问。
2. 用户的原始回答。
3. 回答缺陷分析。
4. AI 重塑后的高分模板。
5. 为什么修改后更好。
6. 可复用的表达框架。

## 6. 故事库模块 (Story Library)

### 6.1 目标
把用户的个人经历沉淀为可复用的面试表达资产。

### 6.2 功能描述

*   从简历中自动抽取项目经历、工作成果、协作案例、失败案例和成长案例。
*   支持用户手动新增故事。
*   每个故事可以关联多个岗位方向和能力标签。
*   同一个故事可以生成不同表达版本：
    *   STAR 结构版
    *   3 分钟讲述版
    *   技术深挖版
    *   业务结果版
    *   管理协作版
    *   英文面试版
*   用户可以从复盘中收藏高分回答，沉淀回故事库。

### 6.3 产品价值
故事库让 JobCopilot 不只是一次性模拟工具，而是能长期积累用户个人表达资产的求职训练系统。

## 7. 题库模块 (Question Bank)

### 7.1 目标
按岗位方向组织高频问题、压力问题和专项练习题。

### 7.2 功能描述

*   按岗位方向查看题库。
*   支持行为面、专业面、HR 面、压力面、英文面等分类。
*   每道题可关联推荐回答结构和用户自己的历史回答。
*   支持收藏、标记掌握程度和加入专项训练。
*   支持 5 分钟快速练习。

### 8.1 目标
帮助用户看到长期进步，而不是只看单次分数。

### 8.2 功能描述

*   查看每次模拟面试的完整报告。
*   按岗位方向统计能力变化。
*   展示薄弱能力、常见表达问题和高频失分点。
*   支持回看历史问答和回答重塑结果。
*   支持从复盘中一键加入故事库或题库。

## 10. AI 架构与多模型协同方案

由于核心大模型擅长文本推理，但文件解析、图像识别、语音识别、语音合成分别需要不同能力，系统建议采用“多模型流水线”。

### 10.1 文件与图片解析管线

*   **路由解析层：** 后端接收文件并判断 MIME Type。
*   **文本类文件：** PDF/Word 可通过 `pdf-parse`、`pdfplumber`、`mammoth` 等工具提取纯文本。
*   **图像类文件：** PNG/JPG/扫描件/截图可调用 OCR 能力，例如 PaddleOCR 或云端 OCR。
*   **统一输出：** 清洗后的 `resume_text`、`jd_text` 作为上下文传入后续分析链路。

### 10.2 实时语音面试管线

*   **ASR：** 前端采集 Web Audio，后端调用 Whisper 或同类 ASR 将用户回答转为文字。
*   **LLM 推理：** 模型根据岗位方向、简历、JD 和历史对话生成下一轮问题或追问。
*   **TTS：** 将 AI 回复文本合成为语音并返回前端播放。
*   **字幕同步：** 每轮语音输出同时返回文字内容和当前考察能力点。

### 10.3 推荐后端接口

#### `POST /api/analysis`
用于准备阶段后的岗位匹配与分析。

输出建议：

```json
{
  "matchScore": 85,
  "strengths": ["架构设计经验突出"],
  "weaknesses": ["缺少 C 端增长案例"],
  "predictQuestions": ["请讲一个你推动复杂项目落地的案例"]
}
```

#### `WS /api/interview/stream`
用于模拟面试双向流。

流程：

1. 客户端推入分块音频。
2. 后端 ASR 转写。
3. LLM 生成追问。
4. TTS 合成音频。
5. 返回音频、字幕、考察能力点。

#### `POST /api/sessions/{id}/review`
用于生成深度复盘。

处理内容：

*   汇总本次多轮对话。
*   计算综合评分。
*   提取最值得改进的回答。
*   生成回答重塑结果。
*   给出下一步训练建议。

## 11. 数据库实体建议

建议使用关系型数据库与 ORM 映射。核心实体包括：

*   **User：** `id`, `name`, `avatar`, `ai_preference_language`, `ai_preference_style`
*   **RoleTrack：** `id`, `user_id`, `name`, `ability_model`, `created_at`
*   **Resume：** `id`, `user_id`, `title`, `raw_text`, `file_url`, `tags`
*   **JobMaterial：** `id`, `user_id`, `role_track_id`, `role_title`, `jd_text`, `status`
*   **InterviewSession：** `id`, `user_id`, `role_track_id`, `resume_id`, `job_material_id`, `match_score`, `strengths`, `weaknesses`, `overall_rating`
*   **InterviewDialog：** `id`, `session_id`, `role`, `content`, `extracted_skill`, `created_at`
*   **Story：** `id`, `user_id`, `title`, `source_dialog_id`, `role_track_ids`, `star_situation`, `star_task`, `star_action`, `star_result`
*   **Question：** `id`, `role_track_id`, `category`, `content`, `difficulty`, `answer_framework`

## 12. UI/UX 体验纲要

### 12.1 HTML/Figma 导出文件定位
`JobCopilot_Figma_Export.html` 是后续 UI 设计的重要参考文件。它提供了当前 v2 的静态视觉稿，包括：

*   桌面端左侧导航。
*   移动端底部导航。
*   首页大卡片式开始入口。
*   岗位方向/职位卡片。
*   简历/JD 上传卡片。
*   故事库 STAR 内容块。
*   设置页 AI 偏好面板。

后续设计应优先参考该 HTML 中已经形成的视觉语言，再逐步调整信息架构。

### 12.2 视觉风格

*   **主背景：** `bg-slate-50` / `#f8fafc`。
*   **背景纹理：** 使用浅色点阵水印 `radial-gradient`，提升工具质感。
*   **卡片：** 大量使用白色卡片、浅描边、轻阴影，避免重阴影。
*   **品牌色：**
    *   深色主按钮与强调区：`slate-900`
    *   主操作色：`indigo-600`
    *   成功与优势状态：`emerald-600` / `emerald-50`
*   **字体：**
    *   大标题可使用 serif 风格，增强高级和权威感。
    *   控件、正文、列表使用系统 sans-serif。

### 12.3 响应式布局标准

**桌面端：**

*   保持 SaaS 左导航，宽度约 `w-72`。
*   主内容区最大宽度控制在 `max-w-5xl`，居中展示。
*   上传、分析、管理类页面可以使用双列或多列布局。

**手机端：**

*   移除侧边栏，使用 fixed bottom bar。
*   底部导航使用毛玻璃或半透明背景 `backdrop-blur-xl`。
*   所有双列/多列布局在移动端转为单列。
*   核心操作按钮靠近底部，适配触屏操作。
*   手机端不删除核心功能，只重排信息层级。

### 12.4 动效与交互反馈

*   使用 `motion/react` 在页面切换、卡片展开和阶段推进时提供轻量动效。
*   卡片进入动画建议统一为 `y: 10 -> 0` 和 `opacity: 0 -> 1`。
*   面试录音状态应使用放大、ring 外发光或轻微脉冲表现。
*   复盘页中“原始回答”和“重塑回答”应有明确视觉区隔，强化改进前后的对比。
*   所有按钮、列表项和卡片操作具备明确 hover、active 和 disabled 状态。

*   岗位方向选择与管理。
